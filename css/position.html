1. Static

  • This is the default position for all elements.
  • Element stays in the page where it naturally comes.
  • You cannot move it using top, left, right, bottom.
  • Real life: A book kept in a shelf in its usual place.
Code:
<div style="position: static; background: lightblue;"> I am static (default). </div>
Output Preview:
I am static (default).

2. Relative

  • Element stays in its normal place.
  • You can move it slightly using top/left.
  • Other items act as if it is still in the old position.
  • Real life: Moving your chair slightly, but others still think you are at the same desk.
Code:
<div style="position: relative; background: orange; top: 10px; left: 20px;"> I am relative (moved 10px down, 20px right). </div>
Output Preview:
I am relative (moved 10px down, 20px right).

3. Absolute

  • Element is removed from normal flow.
  • It positions itself inside the nearest relative parent.
  • If no relative parent, it attaches to the page.
  • Real life: A sticky note inside a notebook, not on the desk.
Code:
<div style="position: relative; background: lightgray; padding: 40px;"> Parent Box <div style="position: absolute; top: 10px; right: 10px; background: red; color: white;"> I am absolute (top-right corner of parent) </div> </div>
Output Preview:
Parent Box
I am absolute (top-right corner of parent)

4. Fixed

  • Element is removed from normal flow.
  • It stays in the same spot on screen even when you scroll.
  • Real life: WhatsApp chat button always at the corner.
Code:
<div style="position: fixed; bottom: 20px; right: 20px; background: green; color: white; padding: 10px;"> I am fixed (stays at bottom-right always) </div>
Output Preview:
I am fixed (stays at bottom-right always)

5. Sticky

Sticky acts like relative first. But when you scroll, it becomes fixed at the top until its parent section ends.

Real-life Example: In an online shopping website, the filter bar or table header stays visible at the top while you scroll down the list.

6. Z-Index

Z-index decides which element is on top when they overlap.
Think of it like stacking papers on a desk. The one with the higher number is on top.

Real-life Example: In a shop, the invoice slip is kept on top of the catalog.
